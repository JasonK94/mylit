# 진행도 추적 기능

FGS v5.4와 TML7에 진행도 추적 기능이 추가되었습니다. 각 메서드의 실행 시간을 기록하고, 다음 실행 시 예상 시간을 제공합니다.

## 기능

1. **전체 예상 시간 표시**: 시작 시 모든 메서드의 캐시가 있으면 전체 예상 시간 표시
2. **메서드별 예상 시간 표시**: 이전 실행 기록을 바탕으로 각 메서드의 예상 소요 시간 표시
3. **실제 시간 측정**: 각 메서드의 실제 실행 시간 기록
4. **즉시 남은 시간 예측**: 첫 번째 메서드 완료 후 즉시 남은 시간 예측 (첫 메서드 기준)
5. **정확한 남은 시간 예측**: 여러 메서드 완료 후 평균 시간을 바탕으로 더 정확한 남은 시간 예측
6. **캐시 저장**: 실행 시간 데이터를 `.fgs_timing_cache.rds`와 `.tml7_timing_cache.rds`에 저장

## 사용 예시

### FGS v5.4 진행도 표시

```
=== FGS v5.4 시작: 총 11개 메서드, 전체 예상 시간: 25.3분 ===

--- Running Method: random_forest [1/11] (예상: 2.3분) ---
✓ random_forest 완료: 145.2초 (2.4분) | 진행: 1/11 | 예상 남은 시간: 24.0분 (첫 메서드 기준)

--- Running Method: lasso [2/11] (예상: 1.8분) ---
✓ lasso 완료: 112.5초 (1.9분) | 진행: 2/11 | 예상 남은 시간: 20.1분 (평균 128.9초/메서드)

...

=== FGS v5.4 완료: 총 11개 메서드, 18.5분 소요 ===
```

### TML7 진행도 표시

```
Training L2 candidate: glm [1/5] (예상: 0.5분, metric=ROC)
✓ glm 완료: 28.3초 (0.5분) | 진행: 1/5 | 예상 남은 시간: 1.8분

Training L2 candidate: ranger [2/5] (예상: 2.1분, metric=ROC)
✓ ranger 완료: 125.7초 (2.1분) | 진행: 2/5 | 예상 남은 시간: 1.2분

...

=== TML7 L2 학습 완료: 총 5개 모델, 4.2분 소요 ===
```

## 캐시 파일

- `.fgs_timing_cache.rds`: FGS v5.4 메서드별 실행 시간 (최근 5회 평균)
- `.tml7_timing_cache.rds`: TML7 L2 메서드별 실행 시간 (최근 5회 평균)

이 파일들은 작업 디렉토리에 저장되며, 다음 실행 시 자동으로 로드됩니다.

## 예상 시간 계산

### 시작 시
- **캐시가 있는 경우**: 모든 메서드의 캐시가 있으면 전체 예상 시간 표시
- **캐시가 없는 경우**: 전체 예상 시간 없음, 각 메서드별로만 표시

### 메서드별 예상 시간
- **첫 실행**: 예상 시간 없음 (실제 시간만 표시)
- **이후 실행**: 최근 5회 실행 시간의 평균값을 예상 시간으로 표시

### 남은 시간 예측
- **첫 번째 메서드 완료 후**: 해당 메서드의 실행 시간 × 남은 메서드 수 (첫 메서드 기준)
- **두 번째 메서드 이후**: 현재까지 완료된 메서드들의 평균 시간 × 남은 메서드 수 (더 정확)
- **에러 발생 시**: 에러가 발생한 메서드도 시간에 포함하여 남은 시간 계산

## 주의사항

- 캐시 파일은 작업 디렉토리에 저장되므로, 다른 디렉토리에서 실행하면 예상 시간이 없을 수 있습니다.
- 데이터 크기나 시스템 성능이 크게 변하면 예상 시간이 부정확할 수 있습니다.
- 캐시 파일 삭제 시 예상 시간 기능이 초기화됩니다.
